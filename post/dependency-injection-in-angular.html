Dependency Injection in Angular

#angular #directive #builtin #frontend #development

Introduction

When you develop a smaller part of your system, like a module or a class, you may need to use features from other classes. For example, you may need an HTTP service to make backend class. Dependency Injection, or DI, is a design pattern and mechanism for creating and delivering some parts of an application to other parts of an application that require them. Angular supports this design pattern and you can se it in your applications to increase flexibility and modularity. In Angular, dependencies are typically services,but they also can be values, such as string or functions. An injector for an application ( create automatically during bootstrap) istantiates dependencies when needed, using a configured provider of the service or value.

Prerequisites

For this tutorial are not required particular prerequisites, but you should know somthing about Dependency Injection in general and how to write some little code in Angular, as in Typescript.
