Basic Testing in Angular

#angular #basics #testing #tutorial #frontend #development

Introduction

Testing your Angular application helps you check that your application is working as you expect. Testing can help reveal bugs before they appear and makes it easy to onboard new developer into the application. Testing can be done in two way: write the test before the actual code or write the test after the code. What's important in general is that the code is broken in individual pieces of code that are small enough to write only one functionality and then it can be tested in a simple way. In this little lesson, you will learn how to put in test different part of the application, like Services, HTTP Requests, Forms and other module of the application.

Prerequisites

For this tutorial is necessary to know about Services, Component, Directive, HTTP requests and the general structure of an Angular app.

Conclusion

Angular makes easy to test all of the aspects of your application: from controller, to services, forms and HTTP. Even testing asynchronous code can be done in a simple way, thanks to the framework.
